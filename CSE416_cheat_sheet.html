📐 Linear Regression
Model Assumption

Relationship between features h(X) and output y is linear (fits a hyperplane).

ŷ = w₀ + w₁x₁ + … + wₙxₙ = wᵀh(X)
Training: Find w̃ (parameters)

Step 1 — Loss function (MSE):

MSE = (1/n) Σ (yᵢ - ŷᵢ)²

Step 2 — Gradient Descent to minimize MSE:

w ← w − η · ∇L(w)
  • η (eta) = learning rate (step size)
  • Repeat until convergence → get ŵ
⚖️ Bias-Variance Tradeoff

Total Error = Bias² + Variance + Irreducible Noise

TermDefinition
BiasGap between true f(x) and average predictor. "Wrong assumptions."
VarianceVariability of predictions for a given point. "Too sensitive to training data."
ModelBiasVarianceProblem
Simple (low complexity)HighLowUnderfitting
Complex (high complexity)LowHighOverfitting

Overfitting signs: Train error ≪ Test/Val error (like Q4: train MSE=1000, test=100k)

Fix overfitting: ↓ complexity, add regularization, get more data, reduce features

Fix underfitting: ↑ complexity, add features

Key rule: Use Lasso for feature selection. Ridge keeps all features but small. (Q5: Ridge ≠ feature selection!)

Lasso path: Higher λ → more weights → 0. Features staying non-zero = most important.

λ (lambda) = regularization strength. Chosen via cross-validation.

🔀 Hyperparameter Selection & Cross-Validation
What are hyperparameters?

Settings not learned from data: model complexity, λ, tree depth, etc.

K-Fold Cross-Validation
  • Split training data into K folds
  • Train on K-1 folds, validate on remaining fold — repeat K times
  • Average validation errors → mean CV error
  • Pick model with lowest mean CV error
Model Selection Rules (Q3)
Error TypeUse for selection?Why
Train ErrorNORisk of overfitting
CV ErrorYES ✓Unbiased estimate of generalization
Test ErrorNONo longer unbiased after peeking

Test set = used once at the very end to estimate real-world error.

📊 Classification Basics

Predicts discrete labels (spam/not spam, approval/rejection, etc.)

Score function & Decision Boundary
Score(x) = wᵀh(x)
  • Score(x) > 0 → predict positive class
  • Score(x) < 0 → predict negative class
  • Score(x) = 0 → decision boundary (the dividing line/hyperplane)
Class Imbalance

High accuracy can be misleading when classes are imbalanced. E.g., 99% not-spam → always predicting not-spam = 99% accuracy but useless!

Learning Curve

Plot of train/val error vs. training size. Used to diagnose bias vs. variance issues.

📉 Logistic Regression
When to Use
  • Binary classification (only two possible outcomes)
  • Features must be independent from each other
  • Want to output probability of class membership
  • Example (Q6): Predict P(heart attack | exercise, sleep)
Sigmoid Function
P(y=+1 | x) = 1 / (1 + e^(−Score(x))) P(y=−1 | x) = 1 − P(y=+1 | x) = 1 / (1 + e^(Score(x)))
Training: Maximum Likelihood

Find weights that maximize the likelihood of the observed data.

Use gradient ascent (going up the hill) with learning rate η.

Worked Example (Slide 15)

Features: h1=count("amazing"), h2=count("ok"), h3=count("bad"), h4=count("not"), h5=count("very")

Score(x) = 2h1 + 4h2 − 3h3 − h4

Sentence: "Tom and Jerry is the very very amazing. It is not bad at all, it is more than ok, it is amazing!"

h1=2, h2=1, h3=3, h4=1, h5=2 (h5 not in model) Score = 2(2)+4(1)−3(3)−1(1) = 4+4−9−1 = −2 P(−|x) = 1/(1+e^Score) = 1/(1+e^−2) ≈ 0.119

P(negative class) = 0.119

Overfitting in Logistic Regression

Fix with regularization (Lasso/Ridge) or early stopping.

🧹 Data Preprocessing — Key Rules
Standardization (Z-score scaling)
  • Scales to mean=0, variance=1
  • Only use when data is in the SAME units
  • Q1: Differing currencies → convert units first, THEN standardize
  • Good when different features have different scales
Feature Selection (Q5)

Use Lasso (L1), NOT Ridge, to reduce features. Sweep λ values — features with non-zero weights = important.

Adding Features (Q4)

If train error ≪ test error → overfitting → reduce features/complexity, do NOT add more features.

🌳 Decision Trees (Q7)
  • No max depth → tree memorizes training data → overfitting
  • Deeper tree ≠ better generalization
  • Fix: Set a max depth (prune) to reduce variance

⚖️ Bias & Fairness
Two Worldviews
WorldviewCore BeliefExample stance
WAE
(We're All Equal)		 Observed disparities = structural bias. Protected group membership should NOT cause task differences. Data Scientist A: remove university & neighborhood — they encode historical bias.
WYSIWYG
(What You See Is What You Get)		 Observed space is a good representation of construct space. Data reflects real-world patterns. Data Scientist B: keep all predictive features; "let the data speak."
Pareto Frontier

Set of models where you cannot improve fairness without hurting accuracy (or vice versa). Shows the tradeoff between performance and fairness.

Model Spaces
  • Construct Space: True underlying qualities (e.g., intelligence, credit-worthiness)
  • Observed Space: Measurable proxies (e.g., SAT score, zip code)
  • WYSIWYG: Observed ≈ Construct
  • WAE: Observed ≠ Construct (bias distorts observations)
Q2 Answer Summary

Scientist A = WAE: Disparities are due to systemic bias → correct for it by removing biased features.

Scientist B = WYSIWYG: Data reflects reality → use all predictive signals, even if correlated with protected attributes.

Core difference: Whether observed group differences are "real" or artifacts of structural inequity.

Gradient Descent Update
w_new = w_old − η · ∇L(w_old)
Regularized Loss
Ridge: L(w) + λ Σwⱼ² Lasso: L(w) + λ Σ|wⱼ|

🗺️ Concept Map / Big Picture
ML Pipeline

1. Data → Clean, standardize (same units!), feature engineer

2. Model → Choose type (linear reg, logistic reg, decision tree…)

3. Train → Minimize loss via gradient descent

4. Validate → K-fold CV to tune hyperparameters (λ, depth…)

5. Evaluate → Use test set once for final unbiased estimate

6. Deploy → Monitor for fairness, distribution shift

Diagnosing Model Problems

Fairness Questions to Ask
  • Does my training data reflect structural bias? (WAE view)
  • Are features proxies for protected attributes?
  • What's my model's impact on different groups (Pareto)?
Standardization Checklist
  • ✅ Same units within a feature? → OK to standardize
  • ✅ Different features with different scales? → Standardize
  • ❌ Mixed units in same feature → Convert first!
💡 Common Exam Traps
  • Ridge ≠ feature selection — Lasso does it
  • Never select model by test error — use CV
  • Low train error + high test error = overfitting — add more features won't help
  • High accuracy ≠ good model (class imbalance!)
  • Score(x)=0 is the decision boundary
  • Logistic regression requires independence of features and binary outcome
  • Gradient ascent for logistic (maximizing likelihood), gradient descent for regression (minimizing loss)
  • Decision tree depth = hyperparameter; unlimited depth → overfitting
  • λ=0 → no regularization; λ→∞ → all weights → 0
