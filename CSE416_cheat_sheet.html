📐 Linear Regression Fundamentals
Core Idea

Find weights that fit a hyperplane through data by minimizing prediction error.

ŷ = w₀ + w₁x₁ + … + wₙxₙ = wᵀh(X)
Training Process

1. Loss (Mean Squared Error):

MSE = (1/n) Σᵢ (yᵢ - ŷᵢ)²

2. Optimize with Gradient Descent:

w ← w − η · ∇L(w)
  • η = learning rate (controls step size)
  • Higher η → faster but may overshoot; lower η → slower but stable
  • Repeat until convergence to get final weights ŵ
⚖️ Bias-Variance Tradeoff

Total Error = Bias² + Variance + Irreducible Noise

Definitions
TermMeaning
BiasSystematic error from wrong assumptions. Model consistently off.
VarianceSensitivity to training data. Small changes in data cause large prediction changes.
Model TypeBiasVarianceProblem
Simple↑ High↓ LowUnderfitting
Complex↓ Low↑ HighOverfitting
Detection & Fixes

Overfitting signature: Train error ≪ Test/Val error (e.g., train MSE=1000, test=100k)

Fix overfitting: Reduce complexity, add regularization, get more data, remove features

Fix underfitting: Increase complexity, engineer better features

🔧 Regularization: Ridge vs. Lasso
Ridge (L2)Lasso (L1)

Weight BehaviorShrinks all toward 0, never exactly zeroCan force weights to exactly 0
Feature Selection?NOYES

Key Insight

For feature selection, ALWAYS use Lasso (L1). Ridge keeps all features but small.

Lasso tip: Higher λ → more weights become zero. Features with non-zero weights = most important.

λ (lambda): Regularization strength, chosen via cross-validation.

🔀 Model Selection via Cross-Validation
What are Hyperparameters?

Settings NOT learned from data: model complexity, λ, tree depth, learning rate, etc.

K-Fold Cross-Validation Process
  • Split training set into K equal folds
  • For each fold: train on K-1 folds, validate on the remaining fold
  • Repeat K times (each fold used for validation once)
  • Average all validation errors → CV error estimate
  • Pick model with lowest mean CV error
Which Error for Selection?
Error TypeUse for Selection?Why
Train ErrorNOOptimistic; doesn't reflect generalization
CV ErrorYES ✓Unbiased estimate; no data leakage
Test ErrorNOContaminated after peeking; use once only at end

Golden Rule: Use CV to select among candidate models. Use test set exactly once for final evaluation.

📊 Classification Essentials
What is Classification?

Predict discrete labels (0/1, spam/ham, approve/reject, etc.)

Score Function & Decision Boundary
Score(x) = wᵀh(x)
  • Score(x) > 0 → predict positive class (+1)
  • Score(x) < 0 → predict negative class (-1)
  • Score(x) = 0 → decision boundary (separator line/plane)
Class Imbalance Trap

High accuracy is misleading with imbalanced data. Example: 99% negative class → always predict negative = 99% accuracy, zero skill!

Fix: Use precision, recall, F1-score instead of accuracy.

Learning Curves

Plot of train/validation error vs. dataset size. Diagnoses bias (curves far apart = variance; curves high together = bias).

📉 Logistic Regression for Classification
When to Use
  • Binary classification (exactly 2 classes)
  • Features are roughly independent
  • Need probability output: P(class | features)
  • Example: P(heart attack | age, exercise, sleep)
Sigmoid Function

Converts score (−∞ to +∞) into probability (0 to 1):

P(y=+1 | x) = σ(Score) = 1 / (1 + e^(−Score)) P(y=−1 | x) = 1 − P(y=+1 | x)

Key property: σ(0) = 0.5 (decision boundary)

Training: Maximum Likelihood

Find weights that maximize likelihood of observed labels.

Use gradient ascent (climbing likelihood hill) with learning rate η.

Difference from regression: Regression uses gradient descent; logistic uses gradient ascent.

Worked Example

Model: Score(x) = 2h₁ + 4h₂ − 3h₃ − h₄ (h₅ omitted, weight=0)

Features: h₁="amazing" count, h₂="ok" count, h₃="bad" count, h₄="not" count

Sentence: "...amazing...ok...not bad..." → h₁=2, h₂=1, h₃=3, h₄=1

Score = 2(2) + 4(1) − 3(3) − 1 = 4 + 4 − 9 − 1 = −2 P(negative) = 1 / (1 + e^(−(−2))) = 1 / (1 + e^2) ≈ 0.119

Interpretation: Model predicts ~12% chance of negative class.

🧹 Data Preprocessing Rules
Standardization (Z-score)

Scale features to mean=0, std=1.

Only when features in compatible units.

Rule: If different units (USD vs EUR) → convert to same unit first, THEN standardize.

Feature Selection

Use Lasso (L1), NOT Ridge. Sweep λ; features with non-zero weights survive → those are important.

Adding Features Myth

If train error ≪ test error → overfitting. Do NOT add features; reduce complexity instead.

Decision Trees
  • Unlimited depth → memorizes training data → severe overfitting
  • Fix: Set max_depth hyperparameter (prune tree)
  • Deeper ≠ better generalization

⚖️ Bias & Fairness Frameworks
Two Worldviews in ML Ethics
WorldviewCore BeliefAction
WAE
We're All Equal		 Disparities = systemic bias in society. Protected attributes encode historical inequity. Remove correlated features; build bias-aware models
WYSIWYG
What You See Is What You Get		 Data reflects true underlying patterns. Use all predictive signals. Keep all features; let data speak; model current reality
Model Spaces
  • Construct space: True underlying reality (e.g., true creditworthiness)
  • Observed space: Measurable proxies (e.g., zip code, credit score)
  • WAE belief: Observed ≠ Construct (measurement bias distorts truth)
  • WYSIWYG belief: Observed ≈ Construct (measurements are accurate)
Pareto Frontier

Set of models where you cannot improve fairness without hurting accuracy (and vice versa). Shows the fundamental fairness-accuracy tradeoff.

Takeaway

There is no "objectively correct" answer. Both worldviews are philosophically valid. The choice depends on:

  • Domain knowledge (is bias real in this context?)
  • Stakeholder values (fairness vs. accuracy priority)
  • Legal/ethical requirements

🗺️ ML Pipeline Overview
Step-by-Step

1. Collect & Clean: Handle missing data, outliers, standardize units

2. Preprocess: Standardize if needed, encode categorical, select features

3. Split: Train (60%), validation (20%), test (20%)

4. Model: Choose algorithm (linear, logistic, tree, etc.)

5. Train: Minimize loss via gradient descent/ascent

6. Tune: Use CV to optimize hyperparameters

7. Evaluate: Test set = final unbiased estimate, use once only

8. Deploy: Monitor for fairness, data drift

Common Pitfalls
  • Ridge ≠ feature selection (only Lasso)
  • Never use test for model selection (CV only)
  • Train ≪ test = overfitting (don't add features!)
  • High accuracy ≠ good model (class imbalance!)
  • Score(x) = 0 is the boundary
  • Logistic requires feature independence
  • Regression: gradient descent↓, Logistic: gradient ascent↑
  • λ = 0 = no regularization; λ → ∞ = all weights → 0
  • Tree depth is a hyperparameter (control overfitting!)
